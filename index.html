Missie 2: De Geheimen van Ultrageluid

Goed gedaan, rekruten! 🎉 De HC-SR04 meet inderdaad afstand. Maar hoe? Laten we dieper in de werking duiken!

Wat is ultrageluid? 🔊

Ultrageluid is geluid met een frequentie hoger dan wat mensen kunnen horen (boven 20 kHz). Onze sensor gebruikt geluid van ongeveer 40 kHz! Dat is zo hoog dat alleen vleermuizen en dolfijnen het kunnen horen! 🦇🐬

Hoe werkt de HC-SR04? 🧐

  1. Uitzenden: De sensor heeft een zender (Trigger) die een korte ultrageluidspuls uitzendt. 📡
  2. Reizen: Deze geluidsgolf reist door de lucht met de snelheid van het geluid (ongeveer 343 m/s bij kamertemperatuur). 🌊
  3. Reflecteren: Wanneer de geluidsgolf een object tegenkomt, kaatst hij terug naar de sensor. 🏓
  4. Ontvangen: De ontvanger (Echo) van de sensor vangt het gereflecteerde geluid op. 📡
  5. Berekenen: De sensor meet de tijd tussen het uitzenden en ontvangen van het signaal. ⏱️

De berekening 🧮

Om de afstand te berekenen, gebruiken we deze formule:

Afstand = (Tijd * Geluidssnelheid) / 2

We delen door 2 omdat het geluid heen en weer reist!

Vraag: Waarom gebruiken we ultrageluid in plaats van normaal hoorbaar geluid?

Het is cooler 😎
Het reist sneller 🚀
Het is nauwkeuriger voor korte afstanden en stoort mensen niet 🎯
Ultrageluid is goedkoper om te produceren 💰

Missie 3: Bouw je Eigen Afstandsdetector

Nu wordt het echt cool! 😎 We gaan onze eigen afstandsdetector bouwen met de HC-SR04. Hier is wat je nodig hebt:

  • 1x HC-SR04 sensor 📡
  • 1x Arduino (Uno of Nano) 🖥️
  • 4x jumper kabels (male-to-male) 🔌
  • 1x breadboard (optioneel, maar handig) 🍞

Volg deze stappen om je detector te bouwen:

  1. Verbind VCC van de sensor met 5V op de Arduino
  2. Verbind GND van de sensor met GND op de Arduino
  3. Verbind Trig van de sensor met pin 9 op de Arduino
  4. Verbind Echo van de sensor met pin 10 op de Arduino

Vraag: Waarom verbinden we de Trig en Echo pinnen met verschillende pinnen op de Arduino?

Het maakt niet uit, je kunt ze met dezelfde pin verbinden 🤷‍♂️
Trig is voor output (Arduino stuurt signaal) en Echo is voor input (Arduino ontvangt signaal) 🔄
Om de sensor sneller te laten werken ⚡
Zodat we meerdere sensoren kunnen aansluiten 🔢

Missie 4: Programmeer je Detector

Geweldig werk, techneuten! 🦸‍♂️🦸‍♀️ Nu gaan we onze detector tot leven brengen met code. Hier is een uitgebreide uitleg van de code:


// Definieer de pinnen
const int trigPin = 9;
const int echoPin = 10;

void setup() {
  // Start seriële communicatie
  Serial.begin(9600);
  // Stel de pinmodes in
  pinMode(trigPin, OUTPUT);
  pinMode(echoPin, INPUT);
}

void loop() {
  // Variabelen voor de duur en afstand
  long duration, distance;

  // Zorg ervoor dat de trigger pin leeg is
  digitalWrite(trigPin, LOW);
  delayMicroseconds(2);

  // Stuur een 10 microseconde puls naar de trigger pin
  digitalWrite(trigPin, HIGH);
  delayMicroseconds(10);
  digitalWrite(trigPin, LOW);

  // Lees de echo pin, en retourneer de reistijd in microseconden
  duration = pulseIn(echoPin, HIGH);

  // Bereken de afstand
  distance = (duration/2) / 29.1; // Snelheid van geluid is 29.1 microseconden per centimeter

  // Print het resultaat naar de seriële monitor
  Serial.print(distance);
  Serial.println(" cm");

  // Wacht even voordat we opnieuw meten
  delay(1000);
}

Upload de code naar je Arduino en open de Seriële Monitor. Je zou nu de afstand moeten zien! 📏✨

Vraag: Waarom delen we de duur door 2 in de berekening van de afstand?

Om de berekening sneller te maken ⚡
Omdat het geluid heen en weer reist, en we alleen de afstand naar het object willen weten 🔄
Omdat geluid zich twee keer zo snel verplaatst als licht 💨
Het is een fout in de code, we zouden niet moeten delen door 2 🐛
